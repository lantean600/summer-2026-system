# Summer 2026 System

这是 2026 暑期学习与科研成果仓库。它的定位不是普通笔记本，而是暑期主控台：记录目标、拆解任务、沉淀产出、复盘进度。

## 核心原则

- 科研项目是主线：暑假结束时要有可展示产物，而不是只有学习时长。
- 数学学习是基础设施：围绕机器学习、科研阅读和实验理解补齐线性代数、概率论、优化。
- 课业学习是保底系统：防止开学后压力集中爆发。
- 作息健康是系统约束：睡眠、运动、复盘不能依赖临时意志力。
- 每天至少留下一个可检查产出：总结、推导、实验记录、代码提交、论文解读、错题归因都可以。

## 仓库结构

```text
.
|-- .github/ISSUE_TEMPLATE/  # GitHub Issue 表单
|-- docs/                    # 系统说明、路线图、GitHub 设置说明
|-- templates/               # 日志、复盘、论文、实验、数学模板
|-- logs/daily/              # 每日记录
|-- reviews/weekly/          # 每周复盘

```

## 每日最小闭环

1. 在 GitHub Issue 里确认今天推进的 1 到 3 个任务。
2. 写一份 `logs/daily/YYYY-MM-DD.md`。
3. 至少产生一个可检查产出。
4. 晚上记录明天第一件事。

## 每周最小闭环

1. 关闭或更新本周 Milestone。
2. 写一份 `reviews/weekly/week-XX.md`。
3. 统计科研、数学、课业、健康四条线的真实进展。
4. 只调整下周系统中真正影响执行的部分。

## 推荐工具分工

- GitHub：任务、阶段、产出、复盘、版本控制。
- Obsidian 或本地 Markdown：草稿、灵感、推导过程、临时笔记。
- Calendar / Google Tasks：起床、学习块、运动、睡前关机等时间约束。
- Zotero：论文 PDF、引用、批注。
- Anki：术语、公式、定理条件、论文核心概念等长期记忆。

不要第一天就把所有自动化搭完。先让每日记录、Issue 推进、周复盘连续跑两周。
